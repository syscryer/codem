# Task: 普通聊天厂商下拉与滚动优化

## Background

普通聊天供应商设置目前在左栏同时展示已配置供应商和常用厂商目录，右栏展示连接配置与模型。厂商目录长期占用纵向空间，并与左栏、右栏各自的滚动容器叠加，形成拥挤和多重滚动体验。

## Objective

将常用厂商目录收口为右侧可搜索下拉，并移除供应商管理内层滚动条

## Scope

In scope:

- 左栏只保留已配置供应商列表和新增入口。
- 左栏厂商图标增大并居中，选中态使用完整圆角矩形胶囊，不再显示左侧蓝色竖线。
- 新增入口明确命名为“新增配置”；同一厂商可以创建多个拥有独立名称、密钥和模型的供应商实例。
- 在右侧供应商配置表单顶部增加可搜索的厂商下拉，显示品牌图标和厂商名称。
- 厂商下拉包含 9 个内置厂商和“自定义”；选择内置厂商后进入其默认渠道与接口类型，选择自定义后保留手工协议和地址配置。
- 渠道、接口类型、API 地址、密钥和模型管理行为保持不变。
- 移除普通聊天供应商设置壳、左右栏和右侧内容区的内层独立滚动，统一由设置主内容区滚动。
- 下拉支持点击外部关闭、Escape 关闭、键盘聚焦和无匹配空状态。
- 全局搜索框聚焦时移除浅色外扩光圈，保留强调色边框和输入光标作为焦点提示。

Out of scope:

- 不调整后端供应商模板契约或内置支持矩阵。
- 不改变已保存供应商、API Key 和模型的持久化方式。
- 不重构普通聊天与 Agent 的独立机制。

## Impact

- Frontend：`AiProviderManagerDialog.tsx` 中厂商选择状态和布局。
- Styling：普通聊天供应商设置的下拉、分栏高度与滚动策略。
- Tests：普通聊天设置源码契约测试、同厂商多实例数据库测试和浏览器交互验收。
